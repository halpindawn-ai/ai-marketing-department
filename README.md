# The AI Marketing Department

Public artifacts and ready-to-use templates for building an AI marketing team. Companion to [The AI Marketing Department Substack](https://theaimarketingdepartment.substack.com), where I write about building an AI demand gen team in public.

I'm Dawn Halpin. I run demand gen at a healthcare SaaS company, and I'm building a team of five AI employees from the inside. This repo is where the public artifacts from that build live.

## What's in here

**[ai-employee-role-spec-template.md](ai-employee-role-spec-template.md)** is the 11-section template I use to scope every AI employee on the team. Copy it, fill in the brackets, build from there.

More artifacts coming as new issues ship: sanitized versions of my actual role specs, real Claude Code skills, and the org chart structure itself.

## Why this exists

Most AI-for-marketers content is tool roundups or theory. I haven't seen anyone writing about the structural layer, the work you do before the build becomes possible. The Substack is where I write about that layer. This repo is where the artifacts live.

If you're building AI into a marketing function and don't know where to start, the role spec template is the place to start. It's the same scaffolding behind every employee on my team.

## How to use the template

1. Copy `ai-employee-role-spec-template.md` and rename it for the employee you're scoping. One employee per file.
2. Work through the eleven sections. Most of mine took 2 to 3 drafts.
3. Write the specs before you draw the org chart. The handoffs in Section 7 are where the cross-employee structure emerges.
4. Use Section 8 (gap analysis) as your build roadmap. Sequence by leverage, not by what's easiest.
5. Build the operating-manual skill from Section 10. That's the skill that auto-loads before any task in this employee's scope.
